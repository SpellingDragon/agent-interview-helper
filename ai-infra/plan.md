# AI Infra 系统工程面试 14 天每日执行清单

> 适用对象：AI Infra / LLM 推理引擎 / 训练平台方向候选人
> 目标：把面试准备从"知道要学什么"变成"每天具体做什么"
> 使用方式：
> 1. 每天按清单执行
> 2. 每天结束做 15 分钟复盘
> 3. 未完成项顺延，但不要打乱主线优先级

---

## 1. 总原则

这 14 天的准备按四条主线推进：

- `主线一`：把项目讲扎实，尤其是推理引擎优化和分布式训练实践
- `主线二`：把推理引擎 / 分布式训练 / CUDA 补成系统认知
- `主线三`：把 C++ 性能 / OS / 网络 / 系统设计这些工程基础补齐
- `主线四`：形成稳定的面试表达，包括自我介绍、项目叙事、问答节奏和 coding 手感

每天建议时间分配：

- `90 分钟`：项目与岗位叙事
- `90 分钟`：推理引擎 / 分布式训练 / CUDA 理论
- `60 分钟`：C++ / OS / 网络基础
- `60 分钟`：算法 / coding
- `15 分钟`：复盘

如果工作日时间紧，最低保底也要做三件事：

- 口述 1 次项目
- 刷 1 组基础题
- 写 1 道 coding

---

## 2. 执行模板

每天统一按下面四步做：

### Step 1. 输入

- 阅读当日主题材料
- 对照前一天复盘结果
- 明确今天的产出物

### Step 2. 输出

- 至少口述 2 次
- 至少写 1 页笔记
- 至少完成 1 次限时题目练习

### Step 3. 验收

- 是否能脱稿回答
- 是否能给出工程 trade-off
- 是否能举自己项目里的例子
- 是否能说出一个失败模式

### Step 4. 复盘

- 今天最卡的 3 个问题是什么
- 哪些回答还是"概念对，表达乱"
- 哪些题会做，但速度还不够
